## New Features

### 🔐 OAuth 2.1 for MCP Connections

Connecting an AI assistant to DevStride just got easier and more secure. You can now authenticate your MCP client with **OAuth 2.1** instead of an API key.

- **No secrets to copy or store** — your client signs you in through the browser, and DevStride only ever issues a short-lived token that the client refreshes automatically
- **The only option that works with web apps** — OAuth 2.1 is required to connect the **Claude.ai** and **ChatGPT** web apps, which can't use a pasted API key
- **Organization scoped** — on first use you pick the organization the connection is scoped to when you sign in

API keys are still supported if you prefer them — your secret is shown only once, when you create the key.

An MCP connection can invoke any tool you can perform in the UI — including destructive ones (archiving boards and items, deleting comments and time entries, bulk-updating items). Read-only scopes are not yet available. Access is limited to the organization you pick when you sign in.

### 🤖 Connect AI (MCP) — Now in the Menu

Connecting an AI client is now a first-class action. A new **Connect AI (MCP)** item in the main menu (under **Help**) opens a setup page with everything you need to connect DevStride to **Claude.ai**, **Claude Code**, **ChatGPT**, **Codex**, **Cursor**, **Copilot**, and **Gemini CLI**.

Each client has step-by-step instructions and a ready-to-paste MCP server URL. For example, to connect **Claude.ai**:

1. Go to **Settings → Connectors → Add custom connector**
2. Enter the **Name** (DevStride) and paste the **MCP Server URL**: `https://api.devstride.com/mcp`
3. Click **Add**, then **Connect** on the DevStride connector and sign in

### 🔑 Admins Can Reset User MFA

Organization Admins can now reset multi-factor authentication for users in their organization. When a user loses access to their authenticator device, an Admin can clear their MFA so they can set it up again — no need to wait on support.
