## ✨ Clearer planning views and a round of display polish

This release adds item numbers to the Gantt table, makes drag-and-drop re-parenting on the Map easier to aim, and cleans up several small display issues customers spotted.

### 🔢 See item numbers in the Gantt table

The Gantt (Roadmap) view's left-hand list now shows each item's number, styled just like the rows on the Map: a small card icon in the item's color and status, then the item number, then the title. That makes it easy to match a bar to the exact item you mean.

A new **Show Item Numbers** switch in the filter bar's **Options** panel turns the numbers on or off. It's **on by default**, and your choice is remembered as part of a saved view — so each stored view keeps whichever setting you gave it.

📚 Learn more: [**Navigating the Gantt table**](/major-modules/track-progress-gantts/navigating-the-table)

### 🧲 Re-parent top-level workstreams by dragging into them

On the Map, dragging a row onto a collapsed parent "spring-loads" it — the parent opens so you can drill in and drop into one of its children. That drill-in now works when you drag a **top-level workstream** (the left-most column), not just items further down.

It only springs open when you're hovering to move *into* a row, so ordinary reordering stays calm, and the hover you need to hold is now shorter and no longer cancelled by tiny cursor movements.

📚 Learn more: [**Using the Map**](/major-modules/workstreams-and-work-items/using-the-map)

### 🧹 Display fixes

A handful of small visual issues are cleaned up:

- **Settings action buttons** — **New Label Group**, **Manage User Labels**, **Manage Dependencies**, **Request trial extension**, and **Delete** now render as solid buttons instead of hollow outlines that looked like tags.
- **Data Model dropdowns** — the collection menus (priorities, tags, custom fields, time estimation) now float as a distinct card, offset from the grid instead of pixel-aligning with it.
- **"New Planning Cadence" button** — the empty-state button is no longer truncated to a narrow sliver; the same fix applies to the Roadmap empty-state button.
- **In-modal calendars** — the date picker now fills the width of its modal instead of rendering narrower than the dialog.
- **"Create with AI" overlay** — its content now fills the full overlay width instead of only the right half.

### 🔄 A quieter refresh

The "recently opened items" list no longer risks a browser storage-quota error on refresh. It now records an item only when you actually open it, keeps the most recent 40, and stores just what's needed to draw each row.

### 🎨 Consistent theme colors

A couple more color touch-ups so the app follows your selected theme:

- **Bulk edit button** — when you multi-select items, the **Edit** button and its selection-count badge now use your workspace's primary theme color instead of a fixed blue.
- **Board card color bars** — on the kanban card view, each card's top color bar again reflects the item's color (they had been showing gray).
