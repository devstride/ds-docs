![Weekly Logs proposed items](/images/weekly-logs/proposed-items.png)

You do not have to add every task yourself. Weekly Logs can suggest them for you.

The **Proposed items** block is a section of the **timesheet grid**, not part of the collapsible toolbar. It only appears when **Team Logs** is **off** (your own timesheet) and you are in **list** or **grouped** view. The multi-user **Team Logs** layout does not include this block.

Use the tabs on the **Proposed items** group header to switch what rows are shown. You can expand or collapse the group like other grid groups.

## Assigned

**Assigned** shows work items that are currently in progress and assigned to you or your team — tasks the system already knows you are working on.

1. Open the **Assigned** tab on **Proposed items**.
2. Review the suggested in-progress rows for the week.
3. Select an item to move it into your timesheet.
4. Enter hours and notes on the row the same way as for tasks you added from search.

If a tab is disabled (dimmed), there is nothing in that bucket for the current week.

## Recent

**Recent** shows items you logged time on last week. If you are working on the same tasks across multiple weeks, this tab surfaces them so you do not have to find them again.

1. Open the **Recent** tab on **Proposed items**.
2. Review items from the previous week.
3. Select an item to carry it forward into this week's timesheet.
4. Log hours and notes on the row where your organization allows it.

## By Person

**By Person** is for when you need to mirror or reference someone else's timesheet.

1. Open the **By Person** tab on **Proposed items**.
2. Click the **+** button on the tab (or follow the **Select People** flow) to choose one or more teammates.
3. Review their logged items for the week, grouped by person.
4. Select an item to add it to your own timesheet and log time.

Useful for shared tasks, pairing sessions, or coordinating time entries across a team. To change someone else's time directly, turn on **Team Logs** and edit on the main multi-user grid when your role allows it.

## Calendar and Calls

- **Calendar** — Microsoft 365 events for the selected week. Sign in with Microsoft when the grid prompts you. Hour cells on these rows stay read-only; you log time by linking an event to a DevStride task (in **list** view), as described in [Logging](/plugins/weekly-logs/logging).
- **Calls** — Teams call history for the week, using the same Microsoft session as **Calendar**.

When you are signed in and **Calendar** or **Calls** is active, **Refresh** and **Sign out** appear on the **Proposed items** header for that data.
