At the end of the month, or whenever reporting is due, you can export your time entries directly from Weekly Logs as a CSV file.

::alert{title="Availability" type="info"}
Export is available in **My Logs** and **Team Logs** mode. It is disabled in **All Orgs** view — switch to a specific organization to export.
::

## Open the Export Dialog

1. Click **Export** in the header.
2. The export dialog opens.
3. Provide a name to the export. It defaults to **Timesheet Export** followed by today's date; You can change this as needed.

## Set a Date Range

Set a **start date** and **end date** for the export.

- Defaults to the **first day of the current month** through **today**.
- The range can span up to **one year**.
- This controls which entries are included, regardless of which week you are currently viewing in the grid.

## Apply Filters (Optional)

Apply the optional filters below to export as desired:

- **Assignees** — include entries for specific team members only.
- **Workstreams** — limit the export to a specific folder or hierarchy path.
- **Custom Fields** — if your organization uses custom fields, select which ones to include as additional columns in the CSV.

## Download

Once everything is set, click **Export**. The file downloads automatically to your device as a CSV — ready to open in Excel or import into any reporting tool.

## What is in the File

Each row in the exported CSV represents one time entry and includes:

| Column | Description |
|--------|-------------|
| Date | The date the time was logged |
| Full Name | The name of the user who logged the entry |
| Workstream Path | The folder hierarchy the task belongs to |
| Task ID | The item number (e.g. I123) |
| Task Name | The title of the task |
| Work Item Type | The item type (e.g. Feature, Story, Bug) |
| Custom Fields | Any custom fields selected during export |
| Hours | Hours logged for that entry |
| Notes | Any description attached to the entry |
